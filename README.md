# FILE-API-SAVER

REST API для хранения пользовательских файлов с использованием S3-совместимого хранилища MinIO.

Проект реализует регистрацию и аутенфикацию пользователей с помощью JWT. Каждый пользователь имеет доступ только к собственным файлам. Метаданные файлов и пользователей зранятся в PostgresSQL, а сами файлы сохраняются в MinIO.

## Возможности

- Регистрация пользователя
- Авторизация пользователя (JWT)
- Загрузка файлов
- Просмотр списка своих файлов
- Получение информации о конкретном файле
- Скачивание файлов
- Удаление своих файлов
- Администратор может просматривать и удалять файлы всех пользователей

## Использeмые технологии

- Go
- PostgresSql
- GORM
- MinIO (s3)
- JWT
- REST API

 ## Запуск проекта 

1. Клонировать репозиторий
 ```bash
git clone https://github.com/svladislav00-qq/FILE-API-SERVER
cd FILE-API-SERVER
```

2. Запустить зависимости

Необходимо запустить: 

-PostgresSQL
-MinIO

После запуска создайте базу данных и настройте переменные окружения. 

3. Настроить `.env`

``` env
PORT=
DATABASE_URL=
USE_SSL=false

JWT_SECRET=

ENDPOINT=
ACCESS_KEY_ID=
SECRET_ACCESS_KEY_ID=
BUCKET=
LOCATION=
```

4. Запустить приложение

```bash
go run cmd/main.go
```

## Авторизация

После входа сервер возаращает JWT-токен.

Каждый защищенный запрос должен содержать заголовок.

```
Authorization: Bearer <token>
```
